ABOUT THE 3D APP

Crafted In Three Dimensions

This 3D Web Application is an academic submission for the Web 3D module Assignment 2026. It presents three luxury fragrance bottles — Azure Lime, Lost Cherry and F* Fabulous — modelled procedurally and rendered in real time inside the browser using Three.js.

Concept & Design Choices

The visual language is inspired by the Tom Ford Private Blend marketing photography: a single product on a polished marble podium, cinematic rim-lighting, deep negative space, and a serif logotype paired with tightly-tracked sans-serif copy. The dark UI lets the bottle's light transmission and gold label become the focal point.

The colour theme reactively follows the active bottle: deep emerald for Azure Lime, oxblood for Lost Cherry, and a near-black palette for F* Fabulous. This was implemented through a data-theme attribute on the body, with CSS custom properties switching the radial background gradient and accent colours.

3D Modelling Approach

  • Geometry — each bottle uses RoundedBoxGeometry for the body and cap, a small CylinderGeometry for the neck, and a planar label sitting just outside the front face. An inner liquid mesh (the fluid the perfume "fills" to) gives the glass bottles depth.
  • Materials — the two glass bottles use MeshPhysicalMaterial with transmission, ior 1.52, attenuation colour and clearcoat — giving a true tinted glass refraction. F* Fabulous uses a matte MeshStandardMaterial to mimic its real-world rubberised finish.
  • Textures — labels are generated at runtime onto an HTML canvas (so the typography is crisp at any resolution) and used as a CanvasTexture. The marble podium is similarly procedural — bezier veining over a radial gradient.
  • Lighting — a soft ambient pass plus a controllable spotlight ("Headlight"), a coloured point-light rim that re-tints per bottle, and a second fill point-light. A PMREM environment is built in-app from a custom shader sky so the glass has realistic specular highlights without fetching an external HDRI.
  • Cameras — a single PerspectiveCamera is animated between named viewpoints (Front, Side, Top, Close-Up) with eased interpolation; OrbitControls hands manual rotation back to the user.

Interactivity

All controls are wired through plain JavaScript event listeners. The Wireframe toggle iterates the bottle group and flips material.wireframe. Spin animates inside the requestAnimationFrame loop. The model gallery and arrow buttons swap bottles by disposing the previous geometry & materials and constructing a new one — so memory does not leak when the user flicks between products.

Going Beyond the Lab — Deeper Understanding

  • Custom GLSL shader sky used to build the PMREM environment map (vertex / fragment shader pair in main.js).
  • Post-processing via EffectComposer + UnrealBloomPass to add a cinematic glow on highlights.
  • Physically-based refractive glass using MeshPhysicalMaterial (transmission, ior, attenuation, clearcoat) — beyond the basic textured material taught in the labs.
  • Procedural canvas-rendered textures (labels, marble veining) so that the build has no external asset dependencies.
  • Reactive theme system driven by CSS custom properties for an immediate brand shift when a new bottle loads.

Accessibility Considerations

  • Semantic landmarks (<header>, <main>, <nav>, <aside>, <footer>).
  • All interactive controls are real <button> elements — keyboard tab-order works without extra ARIA.
  • Contrast ratio of body copy on the dark gradient meets WCAG AA.
  • The viewer has an aria-label; spin is animated but also pausable via the Spin toggle, so users sensitive to motion can stop it.
  • Layout reflows to a single column under 1100 px so the app is usable on tablet & phone.

Technology Stack

  • HTML5 + CSS3 (custom properties, grid, flex)
  • Three.js r160 — MeshPhysicalMaterial, RoundedBoxGeometry, OrbitControls, PMREMGenerator, EffectComposer, UnrealBloomPass.
  • Vanilla ES Modules via importmap — no build step.
