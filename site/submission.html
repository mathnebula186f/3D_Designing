4. Deeper Understanding Statement

This submission goes beyond the lab tutorials in the following ways:

  1. Custom GLSL shader sky used to drive a procedural PMREM environment map for realistic glass reflections, without relying on an external HDRI download.
  2. Cinematic post-processing via EffectComposer + UnrealBloomPass.
  3. Physically-based refractive glass (MeshPhysicalMaterial with transmission, ior, attenuation colour, clearcoat) to produce true light-bending tinted bottles.
  4. A reactive theming system using CSS custom properties driven by a data-theme attribute, so the background gradient, accent colour and rim-light tint all change when a new bottle is selected.
  5. Procedural canvas textures for labels and marble veining, eliminating external texture dependencies and keeping the deployment lightweight.

How testing changed the design

  • The first build had no headlight floor — turning off the spotlight made the bottle disappear. Now ambient drops to a low value rather than zero, so the off-state is moody but still legible.
  • Switching bottles in the original build leaked GPU memory because old geometries were not disposed. Fix: traverse the previous group and dispose geometry, material and material.map before adding the new bottle.
  • The "Top View" camera was directly above the target which caused OrbitControls to gimbal-lock the up-vector. A 0.01-unit z offset avoids the singularity without being visible.
  • The sidebar originally collapsed in an awkward way on tablet — the layout now reflows cleanly to a single column at 1100 px and below.
